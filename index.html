Restricciones de Datos
1 / 9

¿Qué es una Restricción?

Es una regla que se aplica a una columna (o tabla) para limitar el tipo de datos que pueden ser insertados. Son los "guardianes" de tu base de datos.

Aseguran la precisión y fiabilidad de la información.

2 / 9

NOT NULL

Asegura que una columna no puede tener un valor nulo (vacío). Esencial para campos obligatorios.

CREATE TABLE Empleados (
    ID INT NOT NULL,
    Nombre VARCHAR(100) NOT NULL,
    Email VARCHAR(100)
);
3 / 9

UNIQUE

Garantiza que todos los valores en una columna son diferentes entre sí. Perfecto para emails o nombres de usuario.

CREATE TABLE Usuarios (
    UsuarioID INT,
    Username VARCHAR(50) NOT NULL,
    Email VARCHAR(100) NOT NULL UNIQUE
);
4 / 9

PRIMARY KEY (Clave Primaria)

Es la restricción más importante. Identifica de forma única cada registro en una tabla. Es una combinación de `NOT NULL` y `UNIQUE`.

-- Cada tabla debe tener una y solo una Clave Primaria.
CREATE TABLE Productos (
    ProductoID INT PRIMARY KEY,
    Nombre VARCHAR(100) NOT NULL
);
5 / 9

FOREIGN KEY (Clave Foránea)

Crea una relación entre dos tablas, asegurando la integridad referencial. Un pedido no puede existir sin un cliente asociado.

-- El ClienteID en Pedidos debe existir en la tabla Clientes.
CREATE TABLE Pedidos (
    PedidoID INT PRIMARY KEY,
    ClienteID INT,
    FOREIGN KEY (ClienteID) REFERENCES Clientes(ClienteID)
);
6 / 9

CHECK

Asegura que los valores de una columna satisfacen una condición específica. Valida los datos antes de insertarlos.

CREATE TABLE Personas (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Edad INT CHECK (Edad >= 18)
);
7 / 9

Poniéndolo Todo Junto

En la práctica, las tablas se definen combinando múltiples restricciones para una máxima robustez.

CREATE TABLE Productos (
    ProductoID INT PRIMARY KEY,
    SKU VARCHAR(50) NOT NULL UNIQUE,
    Nombre VARCHAR(100) NOT NULL,
    Precio DECIMAL(10, 2) CHECK (Precio > 0),
    CategoriaID INT,
    FOREIGN KEY (CategoriaID) REFERENCES Categorias(ID)
);
8 / 9

Resumen Final

Las restricciones no son opcionales, son la base de un buen diseño de base de datos.

`NOT NULL`: Campo obligatorio.
`UNIQUE`: No se repiten valores.
`PRIMARY KEY`: Identificador único.
`FOREIGN KEY`: Vínculo entre tablas.
`CHECK`: Cumple una condición.
